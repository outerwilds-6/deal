SmartStation 技术架构文档

基于 Python + FastAPI + 计算机视觉的智能无人驿站自取系统

系统定位

单机部署、Web端交互的智能无人驿站，通过人脸识别、视觉条码识别替代人工核验，实现包裹自动入库、刷脸通行、智能取件的全自动化。

核心能力

人脸识别 1:N QR码扫描 实时视频流 WebSocket 推送

开发状态

核心服务已完成测试
FastAPI 后端与前端 HTML 编写中

API 接口文档

后台管理端 /api/backend

方法 路径 功能 参数
POST /users 人员录入 name, phone, file(图片)
GET /parcels 包裹查询 status, skip, limit
GET /logs 日志查看 skip, limit

驿站工作端 /api/station

方法 路径 功能 说明
POST /scan_in 扫码入库 摄像头抓拍 → QR解析 → 自动入库
GET /video_feed 视频流 MJPEG 格式货物监控画面

客户体验端 /api/client

方法 路径 功能 说明
POST /access/entry 刷脸进门 验证通过 → 开门 → 显示待取包裹
POST /access/exit 刷脸出门 检测遗漏 → 更新包裹状态
GET /video_feed 视频流 MJPEG 格式客户摄像头画面

WebSocket 通信 /ws/{client_type}

三端 WebSocket 长连接，client_type 取值：admin、station、client

推送消息格式

{
    "type": "HARDWARE_ACTION",
    "action": "DOOR_OPEN" | "FORGET_ALERT" | "TAKE_PARCEL",
    "data": { "msg": "..." }
}

应用生命周期

1 初始化DB
2 启动摄像头
3 加载人脸引擎
4 构建特征矩阵
5 对外服务 
@asynccontextmanager
async def lifespan(app: FastAPI):
    # 启动阶段
    DatabaseManager.init_db()                    # 初始化数据库
    app_state.camera = get_camera()
    app_state.camera.start()                     # 启动摄像头线程
    app_state.face_recognizer = FaceRecognizer() # 加载 InsightFace
    app_state.scanner = QRScanner()              # 加载扫描器
    UserRepository.get_all_active_faces()        # 构建人脸缓存
    
    yield  # --- 系统运行中 ---
    
    # 关闭阶段
    app_state.camera.stop()                      # 安全释放摄像头

部署与运行

环境要求

  • Python 3.9+
  • CUDA 11.x+ (可选，用于 GPU 加速)
  • 支持 cv2 的摄像头设备（可选，开发用 Dummy 模式）

快速启动

# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动服务
uvicorn main:app --reload --host 0.0.0.0 --port 8000

# 或使用脚本
bash start.sh

架构设计约束

🔌 服务模块化

services 目录下每个功能（人脸、扫码、摄像头）必须是独立的文件夹，高内聚低耦合。

⚙️ 配置隔离

每个服务文件夹内必须包含 constants.py，集中管理阈值、路径、模拟数据等运行常数。

📋 Micro-Doc 机制

每个模块文件夹需包含 .prompt/ 目录，提供同名的 .md 解释文件，采用"职责+接口+示例"格式。

🎯 路由层职责

routers 只负责接收请求和返回响应，所有业务逻辑与 CV 计算必须交由 services 层处理。
