# 文件：test_db.py

**职责**：数据库模块的端到端集成测试，模拟前端所有典型业务请求场景，验证货柜号分配、取件码返回、异常处理等逻辑。

**测试流程覆盖**：
1. 数据库初始化
2. 用户录入（使用 constants 模拟数据）
3. 包裹入库（自动分配柜号并验证柜号唯一性）
4. 重复入库异常测试
5. 客户进门：查询在库包裹并打印货柜号（取件码）
6. 客户取件：逐件更新状态为已取件
7. 客户出门：检查漏拿并写入日志
8. 后台看板：全量包裹列表与联表日志查询
9. 货柜满压力测试：验证 `RuntimeError` 触发

**外部调用示例**：
```bash
python database/test_db.py
```
